REST API Reference

Authentication Endpoints

POST /auth/login

Authenticates a user with email and password credentials. Returns an access token and refresh token pair. The access token is a JWT valid for 15 minutes. The refresh token is an opaque string valid for 30 days.

POST /auth/refresh

Exchanges a valid refresh token for a new access token. The refresh token is consumed and a new one is issued. This rotation prevents replay attacks.

POST /auth/logout

Revokes the current session and invalidates the refresh token. The access token remains valid until its natural expiration but is added to a server-side deny list.

Resource Endpoints

GET /resources

Lists all resources accessible to the authenticated user. Supports pagination via page and per_page query parameters. Default page size is 20, maximum is 100. Results are sorted by creation date descending.

GET /resources/:id

Returns a single resource by its UUID. Returns 404 if the resource does not exist or the user lacks read permission.

POST /resources

Creates a new resource. The request body must include a name field (string, 1-255 characters) and an optional description field (string, max 2000 characters). Returns the created resource with a 201 status code.

PUT /resources/:id

Replaces a resource entirely. All fields must be provided. Returns 404 if the resource does not exist. Returns 403 if the user lacks write permission.

DELETE /resources/:id

Soft-deletes a resource by setting its deleted_at timestamp. The resource is excluded from list queries but can be restored within 30 days. After 30 days, a background job permanently removes it.

Health Endpoints

GET /health/live

Returns 200 if the process is running. Used by Kubernetes liveness probes. Does not check backing services.

GET /health/ready

Returns 200 if the application is ready to serve traffic. Checks database connectivity and cache availability. Used by Kubernetes readiness probes.
