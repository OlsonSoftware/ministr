ministr uses a two-level configuration system: global settings and per-corpus settings.

## Global Configuration

The global config file lives at `~/.ministr/config.toml`. All fields are optional and fall back to sensible defaults.

```toml
# Root data directory (default: ~/.ministr)
data_dir = "~/.ministr"

# Default embedding model for new corpora
default_model = "all-MiniLM-L6-v2"

# Log output format: "pretty" or "json"
log_format = "pretty"

# Window size (tokens) the advisory usage estimate is anchored to
default_context_budget = 100000

[prefetch]
# Whether predictive prefetching is enabled
enabled = true

# Maximum items in the prefetch cache
cache_size = 128

# Number of recent sections for topical prefetch vector
topic_window = 5

# Fine-tuning knobs (all optional — defaults are sensible)
# alpha = 0.3                     # EMA weight on the most recent section
# adaptive_alpha = true            # Auto-tune alpha when the topic stabilises
# speculative = false              # Run prefetch proactively during agent idle
# max_candidates_per_strategy = 5  # Cap per strategy per cycle
```

### Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `data_dir` | path | `~/.ministr` | Root directory for all ministr data |
| `default_model` | string | `"all-MiniLM-L6-v2"` | Embedding model for new corpora (see [supported models](#supported-embedding-models)) |
| `log_format` | string | `"pretty"` | Log format: `"pretty"` or `"json"` |
| `default_context_budget` | integer | `100000` | Window size the advisory usage estimate is anchored to |
| `prefetch.enabled` | boolean | `true` | Enable predictive prefetching |
| `prefetch.cache_size` | integer | `128` | Max prefetch cache entries |
| `prefetch.topic_window` | integer | `5` | Recent sections for topic vector |
| `prefetch.alpha` | float (optional) | auto | EMA weight on the most recent section (0.0-1.0) |
| `prefetch.adaptive_alpha` | boolean (optional) | auto | Auto-tune alpha based on topical hit rate |
| `prefetch.speculative` | boolean (optional) | auto | Run prefetch proactively during agent idle |
| `prefetch.max_candidates_per_strategy` | integer (optional) | auto | Cap per strategy per cycle |

## Project Configuration (`.ministr.toml`)

Create `.ministr.toml` in your project root (or run `ministr init` to generate one). Minimal config:

```toml
[corpus]
# Directories and files to index
paths = [
    "src",
    "docs",
    "README.md",
]

# Glob patterns to exclude
ignore = [
    "*.snap",
    "node_modules",
]
```

ministr auto-discovers `.ministr.toml` by walking up from the working directory.

### `[corpus]` fields

| Field | Type | Default | Description |
|---|---|---|---|
| `corpus.paths` | list of paths | `[]` | Files and directories to index |
| `corpus.ignore` | list of globs | `[]` | Patterns to exclude from indexing (on top of built-in always-ignore lists) |
| `corpus.include` | list of sub-tables | `[]` | External local directories to include — see [Multi-source corpora](#multi-source-corpora) |
| `corpus.git` | list of sub-tables | `[]` | Git repositories to clone and index alongside local paths |
| `corpus.cloud` | list of sub-tables | `[]` | Pre-built `.ministr-index` bundles to fetch instead of re-indexing locally |
| `corpus.model` | string (optional) | — | Per-repo embedding-model override (falls back to global `default_model`) |
| `corpus.dimension` | integer (optional) | — | Target embedding dimension for Matryoshka truncation. Only effective with Matryoshka-capable models like `nomic-embed-text-v1.5` |
| `corpus.rerank_depth` | integer (optional) | `100` | How many coarse candidates to re-score at full dimension when using a Matryoshka model. `0` disables the second-stage rescore. |

### Multi-source corpora

ministr can index more than your project's own files. Three ways to bring in extra sources:

```toml
[corpus]
paths = ["src", "docs"]

# External local directory — e.g., a shared types repo you have checked out elsewhere
[[corpus.include]]
path = "~/Code/shared-types/src"

# Git repository to clone and index
[[corpus.git]]
repo = "https://github.com/owner/repo.git"
branch = "main"                    # optional; defaults to repository default
paths = ["src", "docs"]            # optional sparse-checkout paths

# Pre-built cloud index bundle
[[corpus.cloud]]
url = "https://releases.example.com/my-project.ministr-index"
name = "my-project"                # optional; defaults to URL filename stem
pin_version = "abc123def456"       # optional; skips staleness checks if set
```

### `[[linked]]` — cross-project queries

Linked projects are separate codebases that keep their own index but can be queried from this project. The agent targets them by passing `project: "<label>"` to any query tool (see `ministr_projects` to list available labels).

Unlike `[[corpus.include]]` (which merges external paths into the current corpus), a linked project stays independent — it has its own `.ministr.toml`, its own index, and its own identity.

```toml
[[linked]]
path = "~/Code/shared-types"
label = "shared-types"       # optional; defaults to the final path component

[[linked]]
path = "../api-server"
```

| Field | Type | Required | Description |
|---|---|---|---|
| `path` | string | yes | Absolute or `~`-relative path to the linked project's root |
| `label` | string | no | Human-readable label the agent uses to target this project (defaults to the final component of `path`) |

Cloned repos (via `ministr_clone`) are also accessible as linked projects — the clone result includes the label.

### `[agent]` — custom agent rules

Rules listed under `[agent]` are appended to every generated advisory file (`.claude/rules/`, `.cursor/rules/ministr.mdc`, `.github/copilot-instructions.md`, `AGENTS.md`, etc.) when you run `ministr init`:

```toml
[agent]
rules = [
    "Always run ministr_references before modifying any shared API",
    "Prefer ministr_survey over ad-hoc grep for cross-cutting concerns",
]
```

### Parser Auto-Detection

ministr detects the parser from file extensions:

| Extension | Parser |
|---|---|
| `.md`, `.markdown` | Markdown (comrak) |
| `.html`, `.htm` | HTML (scraper) |
| `.pdf` | PDF (pdf-extract) |
| `.rs`, `.py`, `.ts`, etc. | Tree-sitter (code + symbols) |

## Supported Embedding Models

ministr supports the following local embedding models. Quantized variants (suffix `-q`) use INT8 quantization for faster inference and smaller model files at a slight quality trade-off. Run `ministr list-models` for the authoritative, up-to-date list.

To switch models, set `default_model` in `~/.ministr/config.toml` (or per-corpus `corpus.model` in `.ministr.toml`):

```toml
default_model = "jina-embeddings-v2-base-code"
```

### Choosing a model for your corpus

There is no single best embedding model — the right choice depends on what the
corpus mostly contains:

| Corpus is mostly… | Recommended model | Why |
|---|---|---|
| **Code** | `jina-embeddings-v2-base-code` (768d) | Code-specialized; **measured win** on a text-to-code retrieval bake-off — R@5 0.673 vs 0.635, nDCG@5 0.604 vs 0.550, MRR 0.681 vs 0.604 against `all-MiniLM-L6-v2`. |
| **Prose / docs / markdown** | `all-MiniLM-L6-v2` (384d, default) | The default wins **every** metric on doc-heavy retrieval; the code model *regresses* on prose. |

The trade-off is real and goes both ways: the code model helps code and hurts
prose, so don't set it globally for a mixed or doc-heavy workspace. Two ways to
apply it:

- **All-code workspace** — set it globally in `~/.ministr/config.toml`
  (`default_model = "jina-embeddings-v2-base-code"`); the daemon and the CLI
  both pick it up.
- **One code repo among others** — set `[corpus] model =
  "jina-embeddings-v2-base-code"` in that repo's `.ministr.toml`. The `ministr`
  CLI honors this per-repo override (`corpus.model` > global `default_model`).

Changing a corpus's model changes its vector dimension (here 384 → 768), so the
corpus is **re-indexed** the next time it is built — the index detects the
model/dimension change on load, discards the stale vectors, and rebuilds. No
manual step is required.

## Environment Variables

| Variable | Description |
|---|---|
| `RUST_LOG` | Controls log verbosity via `tracing-subscriber::EnvFilter`. Example: `RUST_LOG=ministr_core=debug,ministr_mcp=info` |
| `MINISTR_BACKEND` | Force the embedding backend: `candle` (Metal GPU on Apple Silicon), `onnx` / `fastembed` (ONNX Runtime). Empty/unset = auto-detect per platform. |
| `MINISTR_DEVICE` | Set to `cpu` to force CPU execution even on a GPU-capable machine (Metal on macOS, DirectML on Windows). |
| `MINISTR_PREFER_QUANTIZED` | `1` to auto-select the `-q` quantized variant of the configured model when one exists. |
| `INSTALL_DIR` | Used by the install script only — overrides the default install location (`$HOME/.ministr/bin`). |
