ministr is currently in a **closed beta**. Distribution is gated through [ministr.ai/beta](https://ministr.ai/beta); the install paths below describe what ships today.

## The beta channel

1. Visit **ministr.ai/beta** and request access with your email (optionally your GitHub login), or sign in with GitHub directly.
2. Once approved, the page shows **Download for macOS** — a signed and notarized `.pkg` built in CI. macOS verifies the signature on first launch.
3. Feedback and access questions: **beta@ministr.ai**.

The installer lays down the desktop app and the `ministr` CLI together. There is no auto-update yet — re-download from the beta page for new builds.

## What a release contains

- **Desktop app**: macOS `.pkg` (Apple Silicon; Intel via Rosetta 2), Windows NSIS installer, Linux `.deb`/`.rpm`/AppImage. The app bundles the CLI as a sidecar.
- **CLI archives**: standalone `ministr` binaries for Linux x64/arm64, macOS Apple Silicon, and Windows x64, published to GitHub Releases with a `SHA256SUMS` manifest.
- **From source**: the CLI is MIT-licensed and builds standalone — `cargo build -p ministr-cli` in the [public repo](https://github.com/OlsonSoftware/ministr) produces a complete local binary with no cloud dependencies.

## Where things live

| Location | What |
|---|---|
| `~/.ministr/` | data directory: per-corpus indexes, config, logs, run history |
| `~/.ministr/bin/` | the canonical CLI install location (`ministr setup` manages PATH) |
| `~/.ministr/config.toml` | optional global configuration |
| `~/.ministr/corpora/<id>/` | per-project index (SQLite + vector index) |
| `~/.ministr/models/` | downloaded embedding models |
| `/usr/local/bin/ministr` | system CLI installed by the macOS `.pkg` |
| `.ministr.toml` | per-project configuration, at your repo root |

`ministr setup` stages the running binary into `~/.ministr/bin`, adds it to PATH across shells, and cleans up legacy install locations. It is idempotent; `--dry-run` previews, `--uninstall` reverses the PATH change.

## Hardware notes

Embeddings run locally. On Apple Silicon the default backend is Candle with native Metal GPU acceleration; on Windows, ONNX with DirectML (any DX12 GPU, CPU fallback); elsewhere, CPU ONNX. You can force a backend with `MINISTR_BACKEND=candle|onnx|fastembed`, force CPU with `MINISTR_DEVICE=cpu`, or prefer quantized model variants with `MINISTR_PREFER_QUANTIZED=1` (2–3× faster on CPU).
