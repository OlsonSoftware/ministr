ministr is a code intelligence MCP server. It gives AI coding agents AST-level understanding of your codebase: hybrid semantic search across code and docs, symbol-level navigation, real reference graphs, and cross-language bridge detection across **13 binding kinds** (Tauri, napi-rs, PyO3, wasm-bindgen, gRPC, HTTP routes, FFI, and more). Symbols are extracted with full AST parsing for **40 languages**, with text-level indexing for dozens more file types (shaders, Vue, Astro, Dockerfiles, assembly). Everything runs on your machine.

## Start where you are

- **You use an AI coding tool and want it to actually know your code** → install the desktop app and follow the [quickstart](/docs/quickstart). The app shows you, in plain words, exactly what your AI sees.
- **You live in the terminal** → [install](/docs/install) the CLI, run `ministr init` in your repo, and you're wired into Claude Code, Cursor, or VS Code in one command. See the [CLI reference](/docs/cli).
- **You operate agents in CI, headless pipelines, or a team deployment** → read [headless & CI](/docs/guides/headless-and-ci) (there is one gotcha you must know) and the [operator docs](/docs/operator/self-hosting).

## What your agent gets

- **[ministr_survey](/docs/tools/survey)** — hybrid semantic + keyword search across code and docs, ranked, session-deduplicated
- **[ministr_symbols](/docs/tools/symbols) / [definition](/docs/tools/definition) / [references](/docs/tools/references)** — find a symbol, read its source, see every caller before touching it
- **[ministr_bridge](/docs/tools/bridge)** — follow a call across Tauri / napi-rs / PyO3 / wasm-bindgen / gRPC / HTTP / FFI boundaries
- **[ministr_impact](/docs/tools/impact)** — the transitive blast radius of a change, with a risk score
- **[ministr_diagnostics](/docs/tools/diagnostics)** — the project's own compilers and linters as structured data
- **[ministr_run](/docs/tools/run)** — recorded, bounded, auditable shell execution

The full surface is 25 tools — see the [tool reference](/docs/tools).

## What you get

The desktop app is a mirror of your agent's knowledge, not a dashboard. It answers three questions without jargon: *is my AI up to date with my code?* (the trust panel), *what exactly does it see?* (the project mirror, down to per-file indexed sections), and *what did it actually do?* (the proof feed — a receipt per tool call, including which files it read before you edited them). See [the desktop app](/docs/desktop-app).

## How it stays honest

Freshness is never guessed from file timestamps: every freshness verdict you see was produced by hashing the working tree. Sessions are tracked so your agent never pays twice for the same content. And the docs you are reading are rewritten against the source — every tool page reflects the parameter schema the server actually ships.
