The `ministr` binary provides subcommands for running the MCP server, indexing corpora, managing projects, and distributing pre-built indexes.

```text
ministr [OPTIONS] [COMMAND]
```

If no subcommand is given, `ministr` runs `serve` with stdio transport â€” the default behavior for MCP clients.

## Commands

| Command | Purpose |
|---|---|
| `serve` | Start the MCP server (stdio or HTTP transport) |
| `index` | Run ingestion synchronously and exit (no MCP server) |
| `init` | Generate `.ministr.toml` and scaffold agent configs |
| `status` | Show daemon status |
| `search` | Search the corpus via the daemon |
| `export` | Export the corpus index to a portable bundle |
| `import` | Import a bundle into the local corpus store |
| `hooks test` | Validate installed agent hooks |
| `setup` | Add ministr to PATH (used by installers and first-run setup) |
| `cloud` | Cloud operator commands (check, demo, license minting/revoking/rotating, SLA pruning) |
| `atlas` | Atlas operator commands (reindex, manifest) |
| `audit` | Audit log operator commands (prune, ensure-partitions, archive) |
| `api-keys` | API key operator commands (flag-stale) |

## Global options

| Flag | Description |
|---|---|
| `-c, --corpus <PATH>` | Corpus source (repeatable): local paths, `https://` URLs, or `github://` URLs |
| `-C, --config <PATH>` | Path to config file (default: `~/.ministr/config.toml`) |
| `-h, --help` | Print help |
| `-V, --version` | Print version |

## `ministr serve`

Starts the MCP server. Default transport is stdio â€” any MCP client can spawn `ministr` as a subprocess and exchange JSON-RPC over stdin/stdout.

```sh
ministr serve                                              # stdio (default)
ministr serve --transport http --host 0.0.0.0 --port 8080  # Streamable HTTP
ministr serve --transport http --oauth                     # with OAuth 2.1
```

| Flag | Description |
|---|---|
| `-t, --transport <TRANSPORT>` | `stdio` (default) or `http` (Streamable HTTP) |
| `--host <HOST>` | Bind host for HTTP transport (default: `127.0.0.1`) |
| `-p, --port <PORT>` | HTTP port (default: `8080`) |
| `--oauth` | Enable OAuth 2.1 authentication (HTTP transport only) |
| `--oauth-issuer <URL>` | OAuth issuer URL (default: `http://<host>:<port>`) |

stdio mode always delegates to the ministr daemon (auto-spawned if not running), keeping the MCP server lightweight.

## `ministr index`

Runs corpus ingestion synchronously and exits. Useful for pre-warming the index, debugging ingestion issues, or running in CI.

```sh
ministr index                                  # index from .ministr.toml
ministr index --corpus ./src --corpus ./docs   # explicit paths
```

ministr indexes on first MCP connection automatically â€” `ministr index` is optional.

## `ministr init`

Generates `.ministr.toml` with auto-detected corpus paths and scaffolds agent configuration files. See [ministr init](/docs/cli/init) for the full reference.

```sh
ministr init                  # non-interactive, auto-detected
ministr init --interactive    # wizard with prompts (short: -i)
ministr init --force          # overwrite existing .ministr.toml
```

## `ministr status`

Shows the ministr daemon status â€” whether it's running, which corpora are registered, ingestion state, and socket path. Requires the ministr desktop app or `ministr-daemon` to be running.

## `ministr search`

One-shot search against the daemon without going through the MCP protocol. Useful for debugging and shell scripts.

```sh
ministr search "authentication" --top-k 5
ministr search "authentication" -k 5     # short form
```

## `ministr export` / `ministr import`

Export the corpus index to a portable `.ministr-index` bundle. Import on another machine to skip re-indexing.

```sh
ministr export -o my-corpus.ministr-index
ministr import my-corpus.ministr-index
```

Session-local data is stripped during export â€” only the corpus content and embeddings are included.

## `ministr hooks test`

Checks each platform's installed hook files and simulates common tool calls so you can see how each one is steered (which built-in tools are declined, and where a shell command is allowed with a hint).

```sh
ministr hooks test
```

## Environment variables

| Variable | Description |
|---|---|
| `RUST_LOG` | Log verbosity (`ministr=debug`, `ministr_core=trace`, etc.) |
| `MINISTR_BACKEND` | Force embedding backend: `candle`, `onnx`, `fastembed`, or empty (auto-detect) |
| `MINISTR_DEVICE` | Force CPU even on GPU-capable machines (`cpu`). Applies to Metal on macOS and DirectML on Windows. |
| `MINISTR_PREFER_QUANTIZED` | Auto-select the quantized `-q` model variant when one exists (`1` / `true`) |

On macOS, when the FastEmbed/ONNX path is selected, ministr always pins the CoreML execution provider to CPU+GPU compute units. Routing to the Apple Neural Engine is intentionally disabled because of a known ~12 GB/batch CoreML/ANE memory leak ([onnxruntime#14455](https://github.com/microsoft/onnxruntime/issues/14455)); there is no env-var knob to enable it.

To override the data directory, set `data_dir` in `~/.ministr/config.toml` (see [Configuration](/docs/configuration)) or pass `-C, --config <PATH>` to point at an alternate config file.

## Operator commands

These commands are used by cloud operators and automated cron jobs. They require `MINISTR_PG_URL` to be set.

### `ministr cloud`

| Subcommand | Purpose |
|---|---|
| `check` | Probe every cloud integration (Postgres, Stripe, GitHub, blob) and print a tick/cross table |
| `demo` | End-to-end runner against a deployed cloud â€” OAuth flow, clone, index, survey |
| `watch` | Stream a corpus's indexing progress to the terminal |
| `generate-license-keypair` | Generate an RSA signing keypair for license JWTs |
| `mint-license` | Sign a license JWT for an Enterprise customer |
| `list-licenses` | Print the issuance audit log |
| `revoke-license` | Append a revocation record to the JSONL revocation list |
| `rotate-license-keys` | Re-mint all in-flight licenses against a new keypair |
| `sla-prune-snapshots` | Delete latency snapshots older than a threshold |

See [License Key Management](/docs/operator/license-mint) for the full license lifecycle.

### `ministr audit`

| Subcommand | Purpose |
|---|---|
| `prune` | Drop audit events older than `--retention-days` (default 90) |
| `ensure-partitions` | Extend quarterly partition surface forward |
| `archive` | Archive a partition to gzipped JSONL and detach from the database |

### `ministr api-keys`

| Subcommand | Purpose |
|---|---|
| `flag-stale` | Flag API keys unused for `--threshold-days` (default 90) and emit audit events |

### `ministr atlas`

| Subcommand | Purpose |
|---|---|
| `reindex` | Re-index every Atlas seed repo |
| `manifest` | Emit the public Atlas manifest as JSON on stdout |
