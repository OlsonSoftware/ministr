```
ministr [OPTIONS] [COMMAND]
```

Global options: `-c, --corpus <PATH|URL>` (repeatable; local paths, `https://`, or `github://` sources) and `-C, --config <PATH>` (global config file, default `~/.ministr/config.toml`).

**Running `ministr` with no command is `ministr serve`** — which is why MCP client configs are just `{ "command": "ministr" }`.

## serve

```bash
ministr serve [--transport stdio|http] [--host 127.0.0.1] [--port 8080] [--oauth]
```

- **stdio** (default): the MCP server your editor spawns. It is a thin proxy to the shared per-machine daemon — no model loading, no index in the proxy process. If no daemon is running it spawns one (detached; it outlives the proxy). Linked projects from `[[linked]]` are addressable via the `project` argument on query tools.
- **http**: a full in-process engine serving MCP (Streamable HTTP at `/mcp`) plus the REST surface — for remote/self-hosted use. `--oauth` enables OAuth 2.1 with scope-gated routes; without it everything is unauthenticated (local dev only). Set `MINISTR_ALLOWED_HOSTS` when serving behind a real hostname.

Before serving, missing agent-config files are scaffolded (create-only — a routine start never rewrites your hook files).

## init

```bash
ministr init [--force] [-i] [--exec-only]
```

Detects the project and writes `.ministr.toml`, MCP client configs (`.mcp.json`, `.cursor/mcp.json`, `.vscode/mcp.json` — merged, never clobbering other servers), and the agent steering scaffold (rules + PreToolUse hooks for Claude Code, Cursor, Windsurf, Copilot; a universal `AGENTS.md`). Advisory files are written once and never overwritten; hook files are healed to the current template. `--force` rewrites an existing `.ministr.toml`. `--exec-only` additionally steers raw Bash toward [recorded execution](/docs/guides/recorded-execution). Codex's user-global config is deliberately not touched.

## index

```bash
ministr index
```

Runs ingestion synchronously and exits — pre-warming for CI or first use. Not required: indexing happens automatically on the first agent connection.

## status / search

```bash
ministr status            # daemon version, uptime, memory, model, per-corpus stats
ministr search "query" -k 10   # quick semantic search from the terminal (requires the daemon)
```

`status` falls back to reading the store directly when the daemon isn't running.

## export / import

```bash
ministr export -o myproject.ministr-index
ministr import myproject.ministr-index
```

A portable, compressed index bundle (content DB with session data stripped, vector index, manifest with model name/dimension and source commit). Build once, ship to ephemeral CI runners or teammates; `import` refuses to overwrite an existing corpus. Bundles are model-stamped — the importing side serves them with the same embedding space.

## hooks test

```bash
ministr hooks test
```

Validates the installed steering: parses each client's hook files, then simulates tool calls (grep/find/Glob should be steered; builds, git, file reads should pass) and prints pass/fail.

## setup

```bash
ministr setup [--dry-run] [--uninstall]
```

Stages the running binary into `~/.ministr/bin`, manages PATH across shells (bash/zsh/fish/nushell/PowerShell and more), and cleans legacy install locations. Idempotent; used by the app's first-launch setup and the install script.

## Environment

See [configuration](/docs/guides/configuration#environment-variables) for the full table (`MINISTR_CORPUS_PATHS`, `MINISTR_BACKEND`, `MINISTR_DEVICE`, `RUST_LOG`, …). Logs always go to stderr — stdout is reserved for the MCP protocol.
