---
title: ministr_diagnostics
description: Run the project's own toolchains and return bounded structured diagnostics.
---

Run the project's own toolchain(s) (cargo, tsc, eslint, ruff, go vet, …, plus
any SARIF-emitting tool) and return bounded, structured diagnostics — file,
range, severity, code, message — errors first, each cross-linked to the
enclosing symbol.

This is the agentic verify step: structured compiler/lint feedback as data,
never raw build logs. Language-agnostic.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `languages` | string[] | — | Optional languages to run (e.g. `rust`, `typescript`, `python`, `go`). Omit to run every detected toolchain. |
| `limit` | number | `100` | Maximum number of diagnostics to return, capped at 500. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `DiagnosticsResponse` with `diagnostics` and `total`. Each diagnostic
carries file, range, severity, code, and message, cross-linked to the
enclosing symbol.

## Notes

- Errors are ordered first.
- SARIF-emitting tools are supported in addition to the built-in toolchain
  integrations.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
