---
title: ministr_toc
description: Structural overview (table of contents) of the indexed corpus.
---

`ministr_toc` returns a structural overview — a table of contents — of the indexed corpus. Use it to orient your agent on an unfamiliar codebase before drilling in, or to list the sections of a single document.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `document_id` | string | — | Filter the listing to one document. |
| `offset` | usize | 0 | Pagination offset into the entries. |
| `limit` | usize | 100 | Maximum entries to return. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `TocResponse` with `corpus_stats`, `roots`, and `entries`. `corpus_stats` carries `documents`, `sections`, `claims`, `offset`, `returned`, and an optional `ingestion_status`.

## Notes

- `ingestion_status` is `"pending"` or `"running"` and only appears when relevant.
- `roots` is only included when the listing is not filtered to a single document (local mode).
- The response is wrapped in the standard envelope: `coherence_alerts`, `indexing_in_progress`/`indexing_message`, and `next_actions`. Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
