This is the complete tool surface as registered by the server. Tools that can't work in a given setup are pruned from `tools/list` automatically (no web fetcher → no `ministr_fetch`; a prose-only corpus → no symbol tools), so an agent never sees a tool it can't use.

Every response shares a common envelope worth knowing: `coherence_alerts` (content you received has changed — re-read it), `indexing_in_progress` (results may be incomplete), and `next_actions` (concrete suggested follow-up calls). Logical failures come back as soft errors that never cancel a parallel batch.
