---
title: ministr_symbols
description: Find code symbols by name, kind, module, or visibility.
---

Find code symbols (functions, structs, traits, etc.) by name, kind, module, or
visibility. Pair with `ministr_definition` for source and `ministr_references`
before modifying.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `query` | string | — | Optional fuzzy, case-insensitive substring match on the symbol name. |
| `kind` | string | — | Optional symbol kind: `function`, `struct`, `trait`, `enum`, `impl`, `const`, `static`, `type`, `mod`. |
| `module` | string | — | Optional module path prefix. |
| `visibility` | string | — | Optional visibility filter: `pub`, `pub(crate)`, `pub(super)`, or `''` (private). |
| `offset` | number | `0` | Pagination offset. |
| `limit` | number | `100` | Maximum number of symbols to return. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `SymbolsResponse` with `symbols`, `total`, and `offset`. Each symbol carries:

- `id`, `name`, `kind`, `file`, `line`, `signature`
- `doc_preview` — the first line of the symbol's doc comment
- `complexity` — cyclomatic complexity (functions only)
- `caller_count` — transitive caller count (omitted in daemon mode)

## Notes

- When exactly one symbol matches, the response emits a `next_actions`
  suggestion to call `ministr_definition` on it.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
