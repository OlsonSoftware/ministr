`ministr_dropped` tells ministr that your agent has dropped previously delivered content from its context. Call it immediately after dropping content you received. This keeps ministr's view of what you still hold accurate — without it, future `ministr_read` calls on the dropped IDs return short "already delivered" stubs instead of the full text.

## Returns

A `DroppedResponse` with `dropped: Vec<String>` (IDs successfully marked as dropped) and `not_found: Vec<String>` (IDs ministr had no delivery record for).

## Notes

- After reporting a drop, a re-`ministr_read` of the same ID returns the full text again rather than a stub.
- Pairs with `ministr_compress`: compress sections first if you want to keep a short referenceable summary, then drop the originals and report them here.
- Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
