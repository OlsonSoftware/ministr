---
title: ministr_related
description: Follow relationship edges from a claim to related claims.
---

`ministr_related` follows relationship edges — `references`, `contradicts`, `depends_on`, `updates` — from a claim to other claims. Use it when one claim's truth depends on another and your agent needs to trace that dependency.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `claim_id` | string | required | Claim ID to start from. |
| `relation_types` | string[] | — | Filter to edge types: `references`, `contradicts`, `depends_on`, `updates`. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `RelatedResponse` with `related: Vec<RelatedClaimResult>`.

## Notes

- Claim IDs come from `ministr_extract` results or survey hits with `resolution: claim`.
- The response is wrapped in the standard envelope: `coherence_alerts`, `indexing_in_progress`/`indexing_message`, and `next_actions`. Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
