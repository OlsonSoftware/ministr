---
title: ministr_bridge
description: Cross-language bridge links across IPC and FFI boundaries.
---

List cross-language bridge links (Tauri commands, NAPI exports, PyO3
functions, FFI, HTTP routes, etc.). Call this before modifying any IPC or FFI
boundary so you see every cross-language call site.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `query` | string | — | Optional substring match on the binding key or symbol name. |
| `bridge_kind` | string | — | Optional kind filter: `tauri_command`, `tauri_event`, `napi`, `wasm_bindgen`, `pyo3`, `http_route`, `ffi`, `cgo`, `jni`, `uni_ffi`, `grpc`, `flutter_channel`, `electron_ipc`. |
| `language` | string | — | Optional language filter. |
| `file_path` | string | — | Optional file path; matches either endpoint of a link. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `BridgeResponse` with `links` and `total`. Each link carries:

- `kind` and `confidence`
- `export` — `binding_key`, `symbol_name`, `file`, `line`, `language`
- `import` — the same fields for the importing side

## Notes

- `file_path` matches a link if either its export or import endpoint lives in
  that file.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
