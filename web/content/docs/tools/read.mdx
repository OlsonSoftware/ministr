`ministr_read` returns the full content of a section by ID. Reach for it after `ministr_survey` points your agent at a section it needs in full. If you only need atomic claims, call `ministr_extract` instead — it's cheaper.

## Returns

Either a full `SectionDetail` (`section_id`, `heading_path`, `text`, `summary`, `claims_available`) or an `AlreadyDeliveredResponse` (`section_id`, `status: "already_delivered"`, `claims_available`).

## Notes

- Three-case dedup/delta behavior:
  1. New content — delivered as full text.
  2. Already delivered and unchanged — a short `already_delivered` stub. If your agent actually dropped the content from context, report it via `ministr_dropped` and re-read to get the full text again.
  3. Already delivered but changed underneath — re-delivered as a delta update.
- Hits on the warm prefetch cache (populated by `ministr_survey`) are served without a backend call.
- The response is wrapped in the standard envelope: `coherence_alerts` (non-empty when underlying content changed since delivery), `indexing_in_progress`/`indexing_message`, and `next_actions`. Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
