Clones a git repository and indexes its content. Sparse checkout is supported
via `paths`, and cached clones are reused. The tool can also be invoked as an
MCP Task.

> Clone a git repository and index its content. Supports sparse checkout. Cached clones are reused.

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `branch` | string | no | Optional branch to clone (defaults to repository default) |
| `paths` | array of string | no | Optional paths for sparse checkout (e.g. ['docs', 'src']). Omit for full checkout. |
| `repo` | string | no | Remote git repository URL to clone (e.g. 'https://github.com/owner/repo.git') |

Annotations: open-world.

## Returns

In local mode, a `CloneResponse` with:

- `files_discovered`
- `files_indexed`
- `sections_extracted`
- `clone_time_ms`
- `index_time_ms`
- `from_cache`
- `dependency_refs_linked`

In daemon-backend mode, the response is the daemon's shape instead:
`{corpus_id, label, linked_toml_updated, ...}`.

## Notes

- Cached clones are reused.
- In daemon-backend mode the tool delegates to the daemon's clone endpoint,
  which clones the repo, registers a new corpus, and appends a `[[linked]]`
  entry to the parent's `.ministr.toml`. The cloned repo becomes a linked
  project that you query with `project: <label>`.
- Invocable as an MCP Task.
