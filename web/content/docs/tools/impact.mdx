---
title: ministr_impact
description: Transitive blast radius of changing a symbol or a git revision range.
---

Compute the transitive blast radius of changing a symbol. Returns every
caller / implementor / importer N levels deep, plus distinct files, distinct
test files, and a low/medium/high risk score. Use before recommending a
non-trivial refactor.

With `range`, the tool runs in diff-aware mode: it resolves the diff's changed
symbols and unions their blast radius.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `symbol_id` | string | — | The symbol whose blast radius to compute. |
| `range` | string | — | Optional git revision range (e.g. `main..HEAD`). Diff-aware mode: resolves the diff's changed symbols and unions their blast radius. Overrides `symbol_id`. |
| `repo_path` | string | server cwd | Work tree for `range` queries. |
| `max_depth` | number | `3` | Traversal depth, clamped to 1–10. |
| `direction` | string | `incoming` | `incoming` (callers, default) or `outgoing` (callees). |
| `tests_only` | boolean | — | With `incoming`, answers "which tests transitively exercise this symbol". |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

An `ImpactResponse` with `impact` and an optional `changed` field. `changed`
appears only for range queries and carries `range`, `changed_files`, and
`changed_symbols` (capped at 200).

## Notes

- Risk heuristic: `score = symbols + 2×files + 3×tests`. High if score > 50 or
  files > 10; Medium if score > 15 or files > 3.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
