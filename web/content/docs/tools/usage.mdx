---
title: ministr_usage
description: An optional, advisory token-usage estimate for the session — internal accounting, not a signal to stop.
---

`ministr_usage` is opt-in. ministr deliberately does **not** put token-usage figures into ordinary tool responses — the per-call numbers are anchored to a configured window, not the model's real context window, and treating them as a limit made agents abandon work early. This tool exists for the rare case where you explicitly want the estimate anyway.

## Parameters

None.

## Response

```json
{
  "total_budget": 100000,
  "estimated_used": 34500,
  "estimated_remaining": 65500,
  "level": "normal",
  "drop_candidates": [],
  "prefetch_metrics": { "hits": 3, "misses": 7 },
  "session_metrics": {
    "total_deliveries": 12,
    "cumulative_tokens_delivered": 34500,
    "total_evictions": 2,
    "cumulative_tokens_evicted": 5200,
    "total_compressions": 1,
    "cumulative_tokens_compressed": 800,
    "total_tokens_saved": 6000,
    "compression_ratio": 0.174,
    "delta_updates": 0,
    "dedup_hits": 3
  },
  "schema_tokens": 4200,
  "tool_count": 20
}
```

### Response fields

| Field | Description |
|---|---|
| `total_budget` | Total context window budget in tokens (configured, not real) |
| `estimated_used` | Estimated tokens currently used |
| `estimated_remaining` | Estimated tokens remaining |
| `level` | Pressure level: `"normal"`, `"elevated"`, or `"critical"` |
| `drop_candidates` | Ranked eviction candidates (empty under normal pressure). Each has `content_id`, `reason`, `tokens_recoverable`, `score`. |
| `prefetch_metrics` | Prefetch cache hit/miss metrics by strategy |
| `session_metrics` | Cumulative token economics — deliveries, evictions, compressions, dedup hits, delta updates |
| `schema_tokens` | Total tokens consumed by MCP tool schemas (descriptions + parameters) |
| `tool_count` | Number of registered tools |
| `coherence_alerts` | Optional — pending coherence alerts when underlying content has changed |
| `elicitation_evicted` | Optional — content IDs evicted via interactive elicitation (when pressure is elevated) |

## Behavior

- **Advisory only.** These figures are anchored to a configured window size, not the agent's real context window. Don't use them to decide you're low on context or to stop working.
- **Not in normal responses.** Other tools never include this; it has to be requested explicitly.
