---
title: ministr_extract
description: Atomic claims from a section, optionally filtered by query.
---

`ministr_extract` returns the atomic claims of a section, optionally filtered by a relevance query. It's cheaper than `ministr_read` when your agent doesn't need the full prose — for example, when checking a specific fact inside a long section.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `section_id` | string | required | Section ID to extract claims from. |
| `query` | string | — | Relevance filter; only claims relevant to the query are returned. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

An `ExtractResponse` with `claims: Vec<ClaimResult>`.

## Notes

- Use `ministr_read` instead when you need the full text of the section.
- The response is wrapped in the standard envelope: `coherence_alerts`, `indexing_in_progress`/`indexing_message`, and `next_actions`. Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
