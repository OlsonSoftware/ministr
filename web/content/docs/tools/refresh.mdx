Checks cached web and git sources for staleness and re-fetches any content that
has changed. Pass a single URL to check one source, or omit `url` to check all
cached sources.

> Check cached web and git sources for staleness and re-fetch changed content.

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `url` | string | no | Optional URL (web) or repo URL (git) to check for staleness. If omitted, checks all cached sources. |

Annotations: idempotent · open-world.

## Returns

A `RefreshResponse` with:

- `urls_checked`, `urls_refreshed`, `urls_unchanged`, `urls_failed`
- `details` — a list of `{url, status}` entries
- `git_repos_checked`, `git_repos_refreshed`, `git_repos_unchanged`, `git_repos_failed`
- `git_details` — a list of `{repo_url, status}` entries

Status values are `"unchanged"`, `"updated"`, or `"failed: <reason>"`.

## Notes

- Git-related fields are omitted from the response when their counts are zero.
- Omitting `url` performs a staleness check across every cached web and git
  source.
