---
title: ministr_references
description: All callers, implementors, and importers of a code symbol.
---

Find all callers, implementors, and importers of a code symbol. Call this
before deleting or significantly modifying any non-trivial public symbol —
zero references means safe to delete.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `symbol_id` | string | — | The symbol ID to look up. Alternative to position addressing. |
| `file` | string | — | Position addressing: file path (use with `line` and `col`). |
| `line` | number | — | Position addressing: 1-based line number. |
| `col` | number | — | Position addressing: 0-based byte column. |
| `ref_kind` | string | — | Optional reference kind filter: `calls`, `implements`, `imports`, `uses`, `bridge`. |
| `through_implementors` | boolean | `false` | Also include callers of the same-named method on co-implementor types (references-including-overrides; bounded). |
| `offset` | number | `0` | Pagination offset. |
| `limit` | number | `100` | Maximum number of references to return. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `ReferencesResponse` with `references`, `total`, and `offset`.

## Notes

- Symbols can be addressed by `symbol_id` or by position (`file` + `line` +
  `col`).
- `through_implementors` widens the result to callers of the same-named method
  on co-implementor types; the expansion is bounded.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
