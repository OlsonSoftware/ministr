`ministr_compress` produces extractive TF-IDF summaries — roughly 60-80% shorter than the originals — for sections your agent wants to keep referenceable without holding their full text. Pair it with `ministr_dropped` after dropping the originals from context.

## Returns

A `CompressResponse` with `summaries`, `total_original_tokens`, `total_compressed_tokens`, and `compression_ratio` (lower is better).

## Notes

- Summarization is always extractive TF-IDF — no MCP sampling capability is required from the client.
- Typical flow: compress sections you've finished reading, drop the originals, then report the drop via `ministr_dropped`.
- The response is wrapped in the standard envelope: `coherence_alerts`, `indexing_in_progress`/`indexing_message`, and `next_actions`. Logical failures return soft errors (`ok: false`, `error_kind`, `message`) and never cancel sibling parallel tool calls.
