---
title: ministr_definition
description: Full source of a code symbol by ID or file position.
---

Get the full source of a code symbol by ID. Call `ministr_references` first if
you intend to modify or delete the symbol. Pass `blame: true` for git
authorship of the symbol's lines.

Symbols can be addressed either by `symbol_id` or by position (`file` +
`line` + `col`), resolved via the occurrence index, LSP-style.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `symbol_id` | string | — | The symbol ID to resolve. Alternative to position addressing. |
| `file` | string | — | Position addressing: file path (use with `line` and `col`). |
| `line` | number | — | Position addressing: 1-based line number. |
| `col` | number | — | Position addressing: 0-based byte column. |
| `blame` | boolean | — | Include git blame: per-author line counts plus the last commit. Degrades gracefully to no blame when unavailable. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

The symbol definition fields at the top level, plus an optional `blame` field
when requested. The source includes 3 lines of surrounding context.

## Notes

- Position addressing (`file` + `line` + `col`) is resolved via the occurrence
  index, LSP-style.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
