---
title: ministr_fetch
description: Fetch a URL from the web and index its content.
---

Fetches a URL from the web and indexes its content into the corpus. The fetcher
tries an `llms.txt` strategy first and falls back to a direct page fetch.

With `depth` greater than 0 it follows same-domain links, optionally constrained
to a path prefix via `path_filter`. The tool can also be invoked as an MCP Task
(task support is optional).

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `url` | string | — | The URL to fetch and index. |
| `depth` | u32 (optional) | `0` | `0` fetches a single page; `1+` follows same-domain links. |
| `max_pages` | number (optional) | `50` | Maximum number of pages to fetch. |
| `path_filter` | string (optional) | — | Path prefix to restrict crawling, e.g. `/docs/`. |

There is no `project` parameter.

## Returns

A `FetchResponse` with:

- `pages_fetched`
- `sections_indexed`
- `claims_extracted`
- `tokens_added`
- `strategy_used`

## Notes

- Tries `llms.txt` first, then falls back to fetching the page directly.
- Returns a soft `not_available` error when the web fetcher, embedder, index,
  or storage are not configured.
- Hidden from `tools/list` when no web fetcher is configured.
- Invocable as an MCP Task.
