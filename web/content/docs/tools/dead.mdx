Find symbols with zero references — candidates for safe deletion. The tool
filters out `pub` symbols, entry points, and trivial helpers. Double-check
with `ministr_references` before deleting, since dynamic dispatch isn't
tracked.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `kind` | string | — | Optional symbol kind filter. |
| `module` | string | — | Optional module filter. |
| `min_lines` | number | `1` | Minimum symbol size in lines. |
| `limit` | number | `50` | Maximum number of symbols to return, capped at 500. |
| `project` | string | — | Linked-project label from `.ministr.toml` `[[linked]]`. Omit for the primary corpus. |

## Returns

A `DeadCodeResponse` with `symbols` and `total`.

## Notes

- Entry points are filtered out: `main`, `_main`, `_start`.
- `pub` symbols and trivial helpers are excluded from results.
- Dynamic dispatch isn't tracked — confirm with `ministr_references` before
  deleting.
- Responses are wrapped in `ToolResponse<T>`, carrying `coherence_alerts`,
  `indexing_in_progress`, and `next_actions`.
- Soft errors (`ok: false`) never cancel sibling parallel calls.
