## Install ministr

The canonical install page lives on the marketing site at **[ministr.ai/install](/install)**. It auto-detects your platform and leads with a double-click installer — macOS `.pkg`, Windows `.exe`, or Linux `.deb` / `.rpm` / `.AppImage`. Every installer adds the `ministr` CLI to your PATH automatically and the app's first-launch **Setup** screen (identical on all three platforms) confirms it worked. A one-line CLI install for headless setups is exposed below it.

This docs page used to hold a static install table; that table drifted out of sync with what we actually ship. The `/install` page reads its asset list dynamically from the release proxy, so it always matches the latest release.

## Configuration

Per-project config: `.ministr.toml` in the project root (created by `ministr init`):

```toml
[corpus]
paths = ["src", "docs", "README.md"]
ignore = ["*.snap", "node_modules"]
```

See [Configuration](/docs/configuration) for full details.
