Agents need to run commands; you need to know what ran. The `ministr_run` tool family replaces fire-and-forget shell-outs with **recorded** execution: every run is captured, bounded, attributed, and auditable — and visible live in the desktop app.

## What a run gives you

- **Bounded capture**: output is captured head+tail with a fixed cap per side, so a runaway command can't exhaust memory. Exact total byte/line counts are kept even when the middle is dropped.
- **A token-lean digest**: the agent gets the exit code plus a digest containing every error line — not a wall of build output. The full log stays available through [ministr_run_logs](/docs/tools/run-logs), which pages **only what the agent hasn't seen yet**.
- **An audit trail**: every run lands in `~/.ministr/exec_runs.db` with command, cwd, status, timing, and session attribution. The desktop app reads the same store.
- **Cross-process control**: in the normal daemon setup, runs execute in the one machine-wide engine — so a run started by your agent can be watched and killed from the app, and survives the agent's own process.
- **Searchable history**: a finished run is rendered as a bounded report and indexed into your corpus. `ministr_survey("what failed last time?")` actually works. The newest 50 reports are retained by default.

## Guardrails

- A run's `cwd` must be inside an indexed corpus root — the engine canonicalizes and enforces this. Your agent can execute where your code is, not anywhere on disk.
- Timeouts: 600s default, 3600s max. Background runs return a `run_id` immediately; poll with [ministr_run_status](/docs/tools/run-status).
- Cancellation kills the whole process group on Unix (on Windows, only the direct child — a documented gap).
- Honest limits: steering an agent toward recorded runs is a workflow improvement, **not a security boundary**. The enforced boundary is the cwd policy above.

## Exec-only mode

```bash
ministr init --exec-only
```

writes a marker that makes the installed Claude Code hooks deny raw `Bash` and redirect to the recorded run family. Delete `.claude/hooks/ministr-exec-only` to reverse it.

## The four tools

| Tool | Purpose |
|---|---|
| [ministr_run](/docs/tools/run) | run a command (foreground or `background: true`) |
| [ministr_run_logs](/docs/tools/run-logs) | page the captured log (delta) or filter with `query` |
| [ministr_run_status](/docs/tools/run-status) | poll status / exit code / duration |
| [ministr_run_kill](/docs/tools/run-kill) | cancel a running run (process-group kill) |
