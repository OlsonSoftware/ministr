## Quick Start

### 1. Install ministr

Pick your platform on **[ministr.ai/install](/install)** — a double-click installer for macOS (`.pkg`), Windows (`.exe`), and Linux (`.deb` / `.rpm` / `.AppImage`). The page auto-detects your OS and leads with the matching installer; a CLI one-liner is available below it for headless installs.

Every installer does the same thing: lays down the desktop app and adds the `ministr` CLI to your PATH. On first launch the app opens a short **Setup** screen — identical on macOS, Windows, and Linux — that confirms the CLI is wired up (with one-click repair if it isn't), then walks you through indexing a project and connecting your agent. No terminal step required.

### 2. Initialize ministr in your project (`ministr init`)

```sh
cd your-project
ministr init
```

This sets up ministr for every major agent platform in your project at once. `ministr init` writes:

- **`.ministr.toml`** — corpus paths and config, auto-detected from project manifests (`Cargo.toml`, `package.json`, `pyproject.toml`)
- **MCP configs** — `.mcp.json` (Claude Code), `.cursor/mcp.json` (Cursor), and `.vscode/mcp.json` (VS Code / GitHub Copilot). Each existing file is merged non-destructively — only the `ministr` key is added.
- **Agent rules** — `.claude/rules/`, `.cursor/rules/ministr.mdc`, `windsurf/rules/ministr.md`, `.continue/rules/ministr.md`, `.github/copilot-instructions.md`, and a platform-neutral `AGENTS.md`. These teach each agent about the ministr tools and their intended workflow.
- **PreToolUse hooks** — `.claude/settings.json`, `.cursor/hooks.json`, `.windsurf/hooks.json`, and `.github/hooks/`. These **block `Grep`, `Glob`, `find`, and shell-pipe search** in agents that support hooks, redirecting the agent to `ministr_survey` / `ministr_symbols` instead. If you don't want the hooks, delete those files after running `init` — the advisory `.md` / `.mdc` rules stay.

<Callout type="info" title="Don't want the agent tool blocking?">
The hook files above are what enforce the "no Grep / no Glob / use ministr instead" policy. They're machine-generated JSON and auto-healed by `ministr init` to the current template. To opt out for a given platform, delete that platform's hook file — the advisory markdown rules still nudge the agent toward ministr tools, but won't block the built-ins.
</Callout>

Advisory markdown files (`.md`, `.mdc`) are created once and **never overwritten** so you can customise them. Hook JSON files are auto-healed — running `ministr init` again brings them back to the current template.

To verify what's installed, run `ministr hooks test` — it simulates common tool calls and reports which would be blocked on which platforms.

Or, if you only want the corpus config without the agent scaffolding, create `.ministr.toml` manually:

```toml
[corpus]
paths = ["src", "docs", "README.md"]
ignore = ["*.snap", "node_modules"]
```

### 3. Configure your MCP client (manual — only if you skipped `ministr init`)

`ministr init` writes the MCP configs for Claude Code, Cursor, and VS Code / Copilot automatically. If you skipped that step or want to add ministr to a client manually, use one of these snippets:

**Claude Code:**

```sh
claude mcp add ministr -- ministr
```

**Cursor** — create `.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "ministr": {
      "command": "ministr",
      "args": []
    }
  }
}
```

ministr auto-discovers `.ministr.toml` by walking up from the working directory — no flags needed.

### 4. Use the tools

Once connected, your agent has access to these tools:

| Tool | Purpose |
|------|---------|
| `ministr_survey` | Semantic search across docs and code |
| `ministr_read` | Read a section by ID (with deduplication & delta delivery) |
| `ministr_extract` | Get atomic claims from a section |
| `ministr_related` | Follow dependency chains between claims |
| `ministr_toc` | Structural overview of the indexed corpus |
| `ministr_symbols` | Search the code symbol index |
| `ministr_definition` | Get full source of a symbol |
| `ministr_references` | Find callers, implementors, importers |
| `ministr_bridge` | Query cross-language bindings |
| `ministr_budget` | Check context budget and get eviction advice |
| `ministr_compress` | Generate compressed summaries for eviction |
| `ministr_evicted` | Signal that content has been dropped |
| `ministr_fetch` | Fetch a web URL into the corpus |
| `ministr_clone` | Clone and index a git repository |
| `ministr_refresh` | Re-fetch sources that have changed upstream |

See the [tool reference](/docs/tools) for parameter schemas and response formats.

### Typical Workflow

```text
ministr_survey("authentication")
  → 5 relevant sections found

ministr_read("src/auth.rs#login")
  → full section text, 847 tokens

ministr_symbols("validate_token")
  → symbol definition with signature and docs

ministr_references("src/auth.rs::validate_token")
  → 3 callers across the codebase
```

Each response includes a `budget_status` showing how much context budget has been consumed, so the agent can make informed decisions about what to read next.
