<Callout type="info" title="Want the desktop app too?">
The signed `.pkg` installer (coming with the 1.0 release) drops both `ministr.app` and the `ministr` CLI in one go on macOS. This page covers the CLI-only and dev-build paths.
</Callout>

## CLI

### Install script (macOS & Linux)

```sh
curl -fsSL https://ministr.app/install.sh | bash
```

The script detects your platform, downloads the matching archive from GitHub Releases, verifies the SHA-256 checksum, and installs `ministr` to `/usr/local/bin` (or `$HOME/.local/bin` without sudo).

### Cargo (from source, requires Rust 1.85+)

Install the latest `main` directly from the repository:

```sh
cargo install --git https://github.com/AlrikOlson/ministr-rs ministr-cli
```

### Homebrew (macOS) — coming with 1.0

A formula lives in the `homebrew/` directory of this repo. Once the `AlrikOlson/homebrew-tap` repo is published, install with:

```sh
brew install AlrikOlson/tap/ministr
```

### crates.io — coming with 1.0

When the workspace is published to crates.io, install with:

```sh
cargo install ministr-cli
```

### Pre-built binaries

Download from [GitHub Releases](https://github.com/AlrikOlson/ministr-rs/releases) — builds available for:

| Platform | Archive |
|----------|---------|
| macOS (Apple Silicon) | `ministr-aarch64-apple-darwin.tar.gz` |
| macOS (Intel) | `ministr-x86_64-apple-darwin.tar.gz` |
| Linux (x86_64) | `ministr-x86_64-unknown-linux-gnu.tar.gz` |
| Linux (aarch64) | `ministr-aarch64-unknown-linux-gnu.tar.gz` |
| Windows (x86_64) | `ministr-x86_64-pc-windows-msvc.zip` |

Each archive has a corresponding `.sha256` checksum file.

## Desktop App (macOS)

End users should grab the pre-built `.pkg` from the releases page — it's signed, notarized, and installs both `ministr.app` and the `ministr` CLI with `PATH` wiring done for you.

Building locally from source (for maintainers):

```sh
just pkg       # signed + notarized build (requires Apple Developer ID)
just pkg-dev   # local testing (no notarization)
```

See `installer/SIGNING-GUIDE.md` for code signing setup.

## Configuration

Per-project config: `.ministr.toml` in the project root (created by `ministr init`):

```toml
[corpus]
paths = ["src", "docs", "README.md"]
ignore = ["*.snap", "node_modules"]
```

See [Configuration](/docs/configuration) for full details.
