---
title: Coherence
description: How ministr keeps the agent's view in sync with the code on disk, so it never reasons off stale context.
---

Coherence keeps the agent's view of your codebase in sync with what's actually on disk. When files change, ministr notices, updates its model, and tells any active session that was holding the old version.

## How it works

While ministr is serving a project it watches the source for changes. When a file is modified, ministr re-reads it and updates the model — incrementally, so the work is proportional to the size of the change, not the size of the codebase. Deleted files are removed from the model.

If a session had already been shown something that just changed, the agent gets a small **coherence alert** on its next tool call:

```json
{
  "coherence_alerts": [
    {
      "section_id": "docs/api.md#rate-limits",
      "change_type": "modified",
      "summary": "Rate limit values updated"
    }
  ]
}
```

The agent can then decide to re-read the changed part or carry on. And if it does re-read something it already had, ministr returns just the lines that changed rather than the whole section again — current context, minimal cost.

## When watching runs

Watching is on whenever `ministr serve` is running. If you only want a one-shot index — no watcher, no server — use `ministr index` instead; it runs once and exits. Per-project watching can be turned off in configuration.
