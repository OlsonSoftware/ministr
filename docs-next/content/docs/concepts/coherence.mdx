---
title: Coherence
description: How ministr detects underlying document changes and notifies active sessions holding stale content — the CPU cache-coherence analog.
---

Coherence is ministr's equivalent of a CPU cache coherence protocol. When the underlying documents change, ministr detects the change and notifies active sessions that hold stale content.

## How It Works

### File Watching

When `watch = true` in the corpus config, ministr monitors source directories for changes using the `notify` crate. When a file is modified:

1. The file is re-parsed and re-indexed. Only changed sections are re-embedded and re-inserted.
2. The session shadow is consulted — if any active session references a changed section, a coherence alert is generated.
3. Deleted files have their sections removed from the index and marked as invalidated.

### Coherence Alerts

Alerts are attached to tool responses when the agent next calls any ministr tool:

```json
{
  "coherence_alerts": [
    {
      "section_id": "docs/api.md#rate-limits",
      "change_type": "modified",
      "summary": "Rate limit values updated"
    }
  ]
}
```

This allows the agent to decide whether to re-read the changed section or continue with potentially stale context.

### Delta Delivery

When the agent reads a section it previously received, and that section has changed, ministr returns a line-level delta instead of the full text. This shows exactly what was added, removed, or modified — minimizing token cost while keeping the agent's context current.

## Incremental Re-Indexing

ministr tracks file hashes to enable incremental updates:

- Only modified files are re-parsed
- Only changed sections within those files are re-embedded
- The HNSW vector index is updated in place (supports deletion and re-insertion)
- Unchanged content retains its existing embeddings and index positions

This makes re-indexing fast even for large corpora — proportional to the size of the change, not the size of the corpus.

## Configuration

File watching is controlled per corpus in your project's `.ministr.toml`:

```toml
[corpus]
paths = ["src", "docs"]
watch = true       # default — enable the file watcher
```

When `watch = false`, coherence is passive — ministr only detects changes when the corpus is explicitly re-indexed (e.g. via `ministr index`).
