title: Core Concepts
description: The six mechanisms ministr uses — session shadow, delta delivery, predictive prefetch, budget management, coherence, and hybrid search.
---

ministr is built around six mechanisms — each one fixes a specific way AI coding agents waste their context window.

| Mechanism | What it does |
|---|---|
| [Session Shadow](/docs/concepts/session-shadow) | Tracks what your agent has already been sent this session |
| [Delta Delivery](/docs/concepts/coherence#delta-delivery) | Sends only the changed lines when a file the agent already saw has been edited |
| [Predictive Prefetch](/docs/concepts/prefetch-engine) | Guesses the agent's next read and has the result ready before it's asked for |
| [Budget & Pressure](/docs/concepts/budget-management) | Tracks context-window fill and auto-compresses responses as it fills up |
| [Coherence Watcher](/docs/concepts/coherence) | Notices file changes on disk and flags previously-delivered content as stale |
| [Hybrid Search](/docs/architecture-deep-dive#the-embedding-stack) | Combines semantic ("meaning") search with keyword search and fuses the rankings |

These mechanisms compose. The session shadow feeds the prefetch engine (knowing what was already delivered informs what to pre-warm), budget pressure decides how much to compress, and the coherence watcher uses the shadow to figure out which sessions hold stale content after a file changes.
