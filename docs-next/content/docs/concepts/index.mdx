title: Core Concepts
description: The ideas behind ministr's code intelligence — hybrid search, session memory, delta delivery, predictive prefetch, and coherence.
---

ministr is built around a few mechanisms. The first is how it understands code; the rest keep delivery efficient so the agent spends its window on the work, not on re-reading.

| Mechanism | What it does |
|---|---|
| [Hybrid Search](/docs/architecture) | Combines semantic ("meaning") search with keyword search and fuses the rankings |
| [Session memory](/docs/concepts/session-shadow) | Tracks what the agent has already been sent this session |
| [Delta delivery](/docs/concepts/coherence) | Sends only the changed lines when a file the agent already saw has been edited |
| [Predictive prefetch](/docs/concepts/prefetch-engine) | Anticipates the next request and has the result ready before it's asked for |
| [Coherence](/docs/concepts/coherence) | Notices file changes on disk and flags previously-delivered content as stale |

These compose. Session memory tells prefetch what the agent already has (so it warms what's actually next), and coherence uses the same memory to know which sessions are holding something that just changed.
