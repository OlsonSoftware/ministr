ministr's job is to understand a codebase well enough that an agent can ask real questions about it. Four ideas carry that; a few more keep delivery efficient.

## Code intelligence

| Concept | What it means |
|---|---|
| [Symbols](/docs/concepts/symbols) | ministr reads code as structure, not text — it knows a definition from a mention |
| [References](/docs/concepts/references) | A real graph of who calls, implements, and imports what — not a regex guess |
| [Cross-language bridges](/docs/concepts/bridges) | It follows a call across a language boundary — the thing grep fundamentally can't do |
| [Hybrid search](/docs/concepts/search) | Meaning-based and keyword search together, at the right level of detail |

## Session efficiency

A quiet layer underneath — a bonus, never the headline.

| Concept | What it means |
|---|---|
| [Session memory](/docs/concepts/session-shadow) | Remembers what the agent has already been shown this session |
| [Predictive prefetch](/docs/concepts/prefetch-engine) | Anticipates the next request and has it ready |
| [Coherence](/docs/concepts/coherence) | Notices when files change and flags anything the agent saw before |

These compose: session memory tells prefetch what's already in hand, and coherence uses the same memory to know which sessions are holding something that just changed — so the agent spends its window on the work, not on re-reading.
