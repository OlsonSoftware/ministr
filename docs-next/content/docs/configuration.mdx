ministr uses a two-level configuration system: global settings and per-corpus settings.

## Global Configuration

The global config file lives at `~/.ministr/config.toml`. All fields are optional and fall back to sensible defaults.

```toml
# Root data directory (default: ~/.ministr)
data_dir = "~/.ministr"

# Default embedding model for new corpora
default_model = "all-MiniLM-L6-v2"

# Log output format: "pretty" or "json"
log_format = "pretty"

# Default context budget in tokens for new sessions
default_context_budget = 100000

[prefetch]
# Whether predictive prefetching is enabled
enabled = true

# Maximum items in the prefetch cache
cache_size = 128

# Number of recent sections for topical prefetch vector
topic_window = 5

# Fine-tuning knobs (all optional — defaults are sensible)
# alpha = 0.3                     # EMA weight on the most recent section
# adaptive_alpha = true            # Auto-tune alpha when the topic stabilises
# speculative = false              # Run prefetch proactively during agent idle
# max_candidates_per_strategy = 5  # Cap per strategy per cycle
```

### Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `data_dir` | path | `~/.ministr` | Root directory for all ministr data |
| `default_model` | string | `"all-MiniLM-L6-v2"` | Embedding model for new corpora (see [supported models](#supported-embedding-models)) |
| `log_format` | string | `"pretty"` | Log format: `"pretty"` or `"json"` |
| `default_context_budget` | integer | `100000` | Token budget for new sessions |
| `prefetch.enabled` | boolean | `true` | Enable predictive prefetching |
| `prefetch.cache_size` | integer | `128` | Max prefetch cache entries |
| `prefetch.topic_window` | integer | `5` | Recent sections for topic vector |
| `prefetch.alpha` | float (optional) | auto | EMA weight on the most recent section (0.0-1.0) |
| `prefetch.adaptive_alpha` | boolean (optional) | auto | Auto-tune alpha based on topical hit rate |
| `prefetch.speculative` | boolean (optional) | auto | Run prefetch proactively during agent idle |
| `prefetch.max_candidates_per_strategy` | integer (optional) | auto | Cap per strategy per cycle |

## Project Configuration (`.ministr.toml`)

Create `.ministr.toml` in your project root (or run `ministr init` to generate one):

```toml
[corpus]
# Directories and files to index
paths = [
    "src",
    "docs",
    "README.md",
]

# Glob patterns to exclude
ignore = [
    "*.snap",
    "node_modules",
]
```

ministr auto-discovers `.ministr.toml` by walking up from the working directory.

### Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `corpus.paths` | list of paths | `[]` | Files and directories to index |
| `corpus.ignore` | list of globs | `[]` | Patterns to exclude from indexing |

### Parser Auto-Detection

ministr detects the parser from file extensions:

| Extension | Parser |
|---|---|
| `.md`, `.markdown` | Markdown (comrak) |
| `.html`, `.htm` | HTML (scraper) |
| `.pdf` | PDF (pdf-extract) |
| `.rs`, `.py`, `.ts`, etc. | Tree-sitter (code + symbols) |

## Supported Embedding Models

ministr supports the following embedding models via `fastembed`. Quantized variants (suffix `-q`) use INT8 quantization for faster inference and smaller model files at a slight quality trade-off.

| Model Name | Dimensions | Notes |
|---|---|---|
| `all-MiniLM-L6-v2` | 384 | Default — fast, general-purpose |
| `all-MiniLM-L6-v2-q` | 384 | Quantized variant |
| `all-MiniLM-L12-v2` | 384 | Slightly higher quality |
| `all-MiniLM-L12-v2-q` | 384 | Quantized variant |
| `bge-small-en-v1.5` | 384 | BAAI small English |
| `bge-small-en-v1.5-q` | 384 | Quantized variant |
| `bge-base-en-v1.5` | 768 | BAAI base English |
| `bge-base-en-v1.5-q` | 768 | Quantized variant |
| `bge-large-en-v1.5` | 1024 | BAAI large English |
| `bge-large-en-v1.5-q` | 1024 | Quantized variant |

To use a quantized model, set `default_model` in `config.toml`:

```toml
default_model = "all-MiniLM-L6-v2-q"
```
