---
title: ministr_related
description: Follow dependency chains between claims — references, contradictions, dependencies, updates.
---

Follow dependency chains between claims. Given a claim ID, returns other claims that reference, depend on, contradict, or update it.

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `claim_id` | string | yes | Claim ID to find related claims for |
| `relation_types` | list of strings | no | Filter by relation type |

### Relation Types

| Type | Description |
|---|---|
| `"references"` | The related claim references or cites this claim |
| `"contradicts"` | The related claim conflicts with this claim |
| `"depends_on"` | The related claim depends on this claim being true |
| `"updates"` | The related claim is a newer version of this claim |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). The `result` payload:

```json
{
  "budget_status": { /* see envelope */ },
  "result": {
    "related": [
      {
        "claim_id": "docs/auth.md#refresh-tokens:c3",
        "text": "Refresh tokens have a 30-day expiry.",
        "relation_type": "references",
        "source_section": "docs/auth.md#refresh-tokens",
        "confidence": 0.82
      }
    ]
  }
}
```

### `result` fields

| Field | Description |
|---|---|
| `related` | List of related claims |
| `related[].claim_id` | The related claim's ID |
| `related[].text` | The related claim's text |
| `related[].relation_type` | How this claim relates to the queried claim |
| `related[].source_section` | Section containing the related claim |
| `related[].confidence` | Detector confidence score (0.0-1.0). Values below ~0.7 are fuzzy matches worth verifying. |

## Behavior

- Enables tracing reasoning chains across documents
- When `relation_types` is omitted, all relation types are returned
- Related claims are discovered via the claim relationship index built during ingestion
