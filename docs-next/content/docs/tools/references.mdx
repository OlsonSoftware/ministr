| Parameter | Type | Required | Default | Description |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). Each reference is a directed edge: `from_*` is the symbol making the reference, `to_*` is the symbol being referenced. The `result` payload:

```json
{
  "budget_status": { /* see envelope */ },
  "result": {
    "references": [
      {
        "from_symbol_id": "sym-src/handlers.rs::handlers::login_handler",
        "from_name": "login_handler",
        "from_file": "src/handlers.rs",
        "from_line": 28,
        "to_symbol_id": "sym-src/auth.rs::auth::validate_token",
        "to_name": "validate_token",
        "to_file": "src/auth.rs",
        "to_line": 42,
        "ref_kind": "calls"
      },
      {
        "from_symbol_id": "sym-tests/auth_test.rs::tests::rejects_expired",
        "from_name": "rejects_expired",
        "from_file": "tests/auth_test.rs",
        "from_line": 15,
        "to_symbol_id": "sym-src/auth.rs::auth::validate_token",
        "to_name": "validate_token",
        "to_file": "src/auth.rs",
        "to_line": 42,
        "ref_kind": "calls"
      }
    ],
    "total": 2,
    "offset": 0
  }
}
```

### `result` fields

| Field | Description |
|---|---|
| `references[].from_symbol_id` | Symbol making the reference |
| `references[].from_name` | Its declared name |
| `references[].from_file` | File containing the reference |
| `references[].from_line` | Line of the reference (1-based) |
| `references[].to_symbol_id` | Symbol being referenced |
| `references[].to_name` | Its declared name |
| `references[].to_file` | File containing the referenced symbol |
| `references[].to_line` | Line where the referenced symbol is defined |
| `references[].ref_kind` | How it's referenced â€” see table below |
| `total` | Total references before pagination |
| `offset` | Offset of the first returned entry within the full result list |

### Reference Kinds

| Kind | Description |
|---|---|
| `calls` | Function/method call site |
| `implements` | Trait implementation |
| `imports` | `use` or `import` statement |
| `uses` | Type reference in signatures or annotations |
| `bridge` | Cross-language binding site (Tauri `invoke`, napi import, PyO3 call, etc.) |

## Behavior

- Includes cross-crate references within the same workspace
- Includes cross-language bridges when applicable (e.g., a Rust `#[tauri::command]` referenced by `invoke()` in TypeScript)
- For finding cross-language bindings explicitly, use `ministr_bridge`
- Use this before modifying or deleting shared code to understand blast radius
