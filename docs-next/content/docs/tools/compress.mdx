---
title: ministr_compress
description: Generate compact extractive summaries of sections the agent wants to keep referenceable without their full text. Returns short summaries with token counts.
---

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `content_ids` | list of strings | yes | Section IDs to generate compressed summaries for |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). The `result` payload:

```json
{
  "budget_status": { /* see envelope */ },
  "result": {
    "summaries": [
      {
        "original_id": "docs/auth.md#jwt-validation",
        "summary": "JWT validation uses RS256 signing with 1-hour expiry. Tokens are verified against the public key from the JWKS endpoint.",
        "original_tokens": 847,
        "compressed_tokens": 95,
        "method": "extractive"
      }
    ],
    "total_original_tokens": 847,
    "total_compressed_tokens": 95,
    "compression_ratio": 0.112
  }
}
```

### `result` fields

| Field | Description |
|---|---|
| `summaries` | List of compressed summaries |
| `summaries[].original_id` | The section or symbol ID that was compressed |
| `summaries[].summary` | Short summary preserving key information |
| `summaries[].original_tokens` | Token count of the original content |
| `summaries[].compressed_tokens` | Token count of the compressed summary |
| `summaries[].method` | Compression method — `"extractive"` (TF-IDF sentence selection, always available), `"abstractive"` (LLM-assisted via MCP sampling, with automatic fallback to extractive on failure), or `"symbol_stub"` (signature + doc comment for code symbols) |
| `total_original_tokens` | Sum of `original_tokens` across all compressed items |
| `total_compressed_tokens` | Sum of `compressed_tokens` across all compressed items |
| `compression_ratio` | Aggregate ratio (`total_compressed / total_original`); lower is better |

## Behavior

- Compression is extractive: selects the most information-dense sentences from the section
- Typically achieves 60-80% token reduction
- Unknown content IDs are silently skipped (not included in the response)
- Use this before evicting content — replace the full section with the compressed summary in the agent's context to preserve the gist while freeing budget
