---
title: ministr_read
description: Read the full text of a section by its hierarchical ID. Returns content with heading path and available claims count.
---

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `section_id` | string | yes | Section ID (e.g. `"docs/auth.md#error-handling"`) |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). The `result` payload varies across three cases:

### New content

```json
{
  "coherence_alerts": [ /* only when content changed; see envelope */ ],
  "result": {
    "section_id": "docs/auth.md#error-handling",
    "heading_path": ["Authentication", "Error Handling"],
    "text": "Full section text...",
    "summary": "How the API surfaces auth failures to the client.",
    "claims_available": 5
  }
}
```

### Already delivered (unchanged)

If the section was already delivered this session and has not changed, ministr returns a **minimal `result`** — just the section ID, `status: "already_delivered"`, and the available-claims count. The text, heading path, and summary are **not** re-shipped, since the agent already received them on the earlier call.

```json
{
  "coherence_alerts": [ /* only when content changed; see envelope */ ],
  "result": {
    "section_id": "docs/auth.md#error-handling",
    "status": "already_delivered",
    "claims_available": 5
  }
}
```

### Already delivered (changed)

If the section was delivered but the underlying document changed, ministr returns a line-level delta showing what was added, removed, or modified.

### `result` fields

| Field | Description |
|---|---|
| `section_id` | The requested section ID |
| `heading_path` | Heading hierarchy (e.g. `["Chapter 3", "Section 3.2"]`). Absent on `already_delivered` fault responses. |
| `text` | Full section content. Absent on `already_delivered` fault responses (the agent already has it). |
| `summary` | Optional — pre-computed section summary when one is available. Absent on `already_delivered` fault responses. |
| `claims_available` | Number of claims that can be extracted with `ministr_extract` |
| `status` | Optional — present as `"already_delivered"` on a re-read fault; absent on a fresh delivery |

## Behavior

- Handles three cases: new content, unchanged re-request (returns the `already_delivered` stub), changed re-request (returns a line-level delta)
