Extract atomic claims from a section, optionally filtered by relevance to a query. Claims are single factual statements that can stand alone.

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `section_id` | string | yes | Section ID to extract claims from |
| `query` | string | no | Optional query to filter claims by relevance |

## Response

### With `query` — ranked by relevance

```json
{
  "claims": [
    {
      "claim_id": "docs/auth.md#jwt-validation:c1",
      "text": "JWT tokens use RS256 signing algorithm.",
      "relevance": 0.92
    },
    {
      "claim_id": "docs/auth.md#jwt-validation:c2",
      "text": "Token expiry is set to 1 hour by default.",
      "relevance": 0.85
    }
  ],
  "budget_status": { ... }
}
```

### Without `query` — document order, no relevance field

```json
{
  "claims": [
    { "claim_id": "docs/auth.md#jwt-validation:c1", "text": "JWT tokens use RS256 signing algorithm." },
    { "claim_id": "docs/auth.md#jwt-validation:c2", "text": "Token expiry is set to 1 hour by default." }
  ],
  "budget_status": { ... }
}
```

### Response Fields

| Field | Description |
|---|---|
| `claims` | List of extracted claims |
| `claims[].claim_id` | Claim ID for use with `ministr_related` |
| `claims[].text` | The atomic factual statement |
| `claims[].relevance` | Optional — relevance to the query (0.0-1.0). Present only when a `query` was supplied; omitted otherwise. |
| `budget_status` | Budget snapshot — present when `session_id` was provided |

## Behavior

- When `query` is provided, claims are ranked by embedding similarity to the query
- When `query` is omitted, all claims from the section are returned in document order
- Claims are the highest-resolution unit in ministr — use them for surgical precision
- Each claim ID can be passed to `ministr_related` to follow dependency chains
