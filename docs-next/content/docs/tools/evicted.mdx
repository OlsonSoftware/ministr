---
title: ministr_evicted
description: Signal that content IDs were dropped from the agent's context. Keeps ministr's view of what the agent still has accurate.
---

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `content_ids` | list of strings | yes | Content IDs the agent has dropped from its context |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). The `result` payload:

```json
{
  "coherence_alerts": [ /* only when content changed; see envelope */ ],
  "result": {
    "evicted": ["docs/setup.md#prerequisites", "docs/intro.md#overview"],
    "not_found": ["unknown-id"]
  }
}
```

### `result` fields

| Field | Description |
|---|---|
| `evicted` | Content IDs ministr no longer considers delivered this session |
| `not_found` | Content IDs ministr had no record of delivering |

## Behavior

- Tells ministr the agent no longer has this content, so a later `ministr_survey` won't filter it out as "already delivered"
- Keeps ministr's session memory accurate — without these signals it relies on a rough estimate that can drift over long sessions
- This is the agent's way of correcting that estimate explicitly
