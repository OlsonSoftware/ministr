---
title: ministr_evicted
description: Signal that content IDs were dropped from the agent's context. Keeps ministr's view of what the agent still has accurate.
---

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `content_ids` | list of strings | yes | Content IDs that have been evicted from the agent's context window |

## Response

Wrapped in the [standard response envelope](/docs/tools#response-envelope). The `result` payload:

```json
{
  "budget_status": { /* see envelope */ },
  "result": {
    "evicted": ["docs/setup.md#prerequisites", "docs/intro.md#overview"],
    "not_found": ["unknown-id"]
  }
}
```

### `result` fields

| Field | Description |
|---|---|
| `evicted` | Content IDs successfully removed from the session shadow |
| `not_found` | Content IDs that were not in the session shadow |

## Behavior

- Removes the specified content from the session shadow
- Reduces the estimated token usage in the budget tracker
- Future `ministr_survey` calls will no longer filter out evicted content
- Improves the accuracy of window estimation — without explicit signals, ministr relies on heuristic estimation which may drift over long sessions
- This is the agent's way of correcting ministr's window estimate, analogous to an explicit cache invalidation
