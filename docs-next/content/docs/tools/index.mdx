ministr exposes MCP tools that give the agent code intelligence and explicit control over retrieval. The tools cover semantic search, code navigation, cross-language bridges, session efficiency, and multi-source corpus ingestion.

## Response envelope

Every MCP tool returns a `ToolResponse<T>` envelope. The tool-specific payload lives under `result`; everything else is wrapper metadata that the agent can use to track session state without re-querying.

```json
{
  "budget_status": {
    "tokens_used": 34500,
    "tokens_remaining": 65500,
    "pressure_level": "normal",
    "utilization": 0.345
  },
  "coherence_alerts": [ /* present only when underlying content has changed */ ],
  "indexing_in_progress": true,
  "indexing_message": "Checking 12/42 files",
  "eviction_recommendations": [ /* advisory; present when the configured-window estimate runs high */ ],
  "result": {
    /* tool-specific payload ‚Äî shape varies per tool */
  }
}
```

| Wrapper field | Always present? | Description |
|---|---|---|
| `budget_status` | ‚úÖ always | Advisory token-usage estimate (`tokens_used`, `tokens_remaining`, `pressure_level`, `utilization`) ‚Äî internal accounting, anchored to the configured window, not a stop signal |
| `coherence_alerts` | only when non-empty | Notifications for sections whose underlying content changed since last delivery |
| `indexing_in_progress` | only when `true` | Background corpus ingestion is still running |
| `indexing_message` | only when present | Human-readable ingestion status (e.g. `"Checking 12/42 files"`) |
| `eviction_recommendations` | only when the window estimate runs high | Advisory list of content ministr suggests dropping first ‚Äî same shape as the `eviction_candidates` field in [`ministr_budget`](/docs/tools/budget) |
| `result` | ‚úÖ always | Tool-specific payload ‚Äî see each tool's reference page for its shape |

The per-tool reference pages below show the full envelope in their `Response` examples for clarity. Field tables describe the `result` payload (shape varies per tool) and reference this envelope for the wrapper fields.

## Typical Workflow

```text
survey ‚Üí read ‚Üí extract ‚Üí related
  ‚Üì                          ‚Üì
budget ‚Üê compress ‚Üê evicted
```

1. **Survey** to orient ‚Äî find relevant areas of the corpus
2. **Read** specific sections identified by the survey
3. **Extract** claims for surgical precision
4. **Related** to follow reasoning chains
5. **Budget** for an advisory token-usage estimate (optional)
6. **Compress** sections into compact summaries when useful
7. **Evicted** to tell ministr what was dropped (improves tracking accuracy)
