ministr exposes MCP tools that give the agent code intelligence and explicit control over retrieval. The tools cover semantic search, code navigation, cross-language bridges, session efficiency, and multi-source corpus ingestion.

## Search & retrieval

|---|---|---|
| [`ministr_survey`](/docs/tools/survey) | Semantic search across docs and code | 100-300 |
| [`ministr_read`](/docs/tools/read) | Read a section in full | 200-2000 |
| [`ministr_extract`](/docs/tools/extract) | Pull specific claims | 50-500 |
| [`ministr_related`](/docs/tools/related) | Follow claim relationships | 50-300 |

## Response envelope

Every MCP tool wraps its answer in a small envelope. The tool-specific payload is always under `result`; the other fields appear only when they apply and carry actionable signals the agent can use without re-querying.

```json
{
  "coherence_alerts": [ /* present only when underlying content changed */ ],
  "indexing_in_progress": true,            /* present only while indexing */
  "indexing_message": "Checking 12/42 files",
  "next_actions": [ /* present only when ministr has concrete suggestions */ ],
  "result": {
    /* tool-specific payload — shape varies per tool */
  }
}
```

| Wrapper field | When present | Description |
|---|---|---|
| `result` | always | Tool-specific payload — see each tool's reference page for its shape |
| `coherence_alerts` | only when non-empty | Sections whose underlying content changed since they were last delivered |
| `indexing_in_progress` | only when `true` | Background indexing is still running, so results may be partial |
| `indexing_message` | only when present | Human-readable indexing status (e.g. `"Checking 12/42 files"`) |
| `next_actions` | only when non-empty | Concrete suggested follow-up calls (e.g. re-read a changed section) |

ministr does not inject token-usage figures into tool responses; if you want an advisory estimate, call [`ministr_budget`](/docs/tools/budget) explicitly.

## Typical workflow

Code intelligence first; the prose tools and session aids fall in around it.

```text
survey ──▶ symbols ──▶ definition ──▶ references ──▶ bridge
   │                                                   (cross-language)
   └──▶ read ──▶ extract ──▶ related        (docs & prose)
```

1. **Survey** to orient — find the relevant area by meaning
2. **Symbols / Definition** — jump to the exact function, type, or trait
3. **References** — see real callers, implementors, importers
4. **Bridge** — follow a call across a language boundary
5. **Read / Extract / Related** — pull prose sections, atomic claims, and follow links

`compress` and `evicted` are optional session-efficiency aids — they help ministr keep delivery lean and are never required to get an answer.
