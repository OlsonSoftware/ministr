Get a structural overview of the indexed corpus — documents, section counts, and ingestion state.

## Parameters

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `document_id` | string | no | — | If provided, return the TOC for a single document |
| `limit` | integer | no | — | Maximum number of entries |
| `offset` | integer | no | 0 | Pagination offset |

## Response

```json
{
  "entries": [
    {
      "id": "src/auth.rs",
      "title": "src/auth.rs",
      "kind": "document",
      "depth": 0,
      "children": 12,
      "source_path": "src/auth.rs"
    },
    {
      "id": "src/auth.rs#validate_token",
      "title": "validate_token",
      "kind": "section",
      "depth": 1,
      "children": 0,
      "source_path": "src/auth.rs"
    }
  ],
  "total": 219
}
```

### Response Fields

| Field | Description |
|---|---|
| `entries[].id` | Content ID — pass to `ministr_read` (sections) or `ministr_toc` with `document_id` for deeper drill-down |
| `entries[].title` | Display title or heading |
| `entries[].kind` | `"document"`, `"section"`, or `"symbol"` |
| `entries[].depth` | Nesting depth (0 = top-level document) |
| `entries[].children` | Number of child sections directly under this entry |
| `entries[].source_path` | Optional — source file path, omitted when not applicable |
| `total` | Total number of entries — useful for paginating with `offset` / `limit` |

## Behavior

- Without `document_id`, returns top-level entries for every document in the corpus (one entry per document)
- With `document_id`, returns that document's section hierarchy
- Use for initial orientation when connecting to an unfamiliar corpus
- To check whether indexing is still in progress, poll the daemon's `/activity` endpoint or watch the desktop app's Overview — not this tool
